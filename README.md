
# Prime Oasis - LCM法による素数ギャップ内の素数発見

大きな素数ギャップの中から素数を発見する革新的なアルゴリズムの実装

## 概要

砂漠にはごく稀にオアシスと呼ばれる水源がある。素数ギャップが大きいとその区間を素数砂漠と言ったりする。素数ギャップの中には中央付近に素数が存在する事がある。そんな砂漠の中のオアシスのような素数を探すプログラムである。

## LCM法について

このプロジェクトは、**LCM法**（最小公倍数法）を用いた新しいアプローチを実証します。

### 発見の経緯

整数の約数を観察していたところ、1,2,3,...と壁のように連なっている箇所があることに気づきました。例えば、1,2,3,4,5,6という約数の連なりは60の位置で現れます。これは最小公倍数 lcm(1,2,3,4,5,6) = 60 の位置でした。60の倍数である120でも同様に、約数として1,2,3,4,5,6という連なりが観察できます。

### LCM法の原理

LCM法は素数そのものを特定するのではなく、素数ギャップの位置を特定します。例えるなら、素数を池の中の魚とした場合、LCMは投網のようなものです。投網を投げても必ず魚が取れるわけではありませんが、従来の消去法とは異なる積極的なアプローチで探索を行います。

lcm(1,2,3,...,n) とその倍数の位置では、1からnまでの全ての数が約数として「壁のように連なる」ため、その周辺は±1を除き合成数が密集します。この性質を利用して、素数ギャップの位置とサイズを正確に特定しつつ素数になり得る場所も掌握します。

### マトリョーシカ構造

LCM法では、複数の層を入れ子にする「マトリョーシカ構造」を採用しています。

- **第1層**: lcm(1,2,3,...,n) で大きな素数ギャップを特定
- **第2層**: 第1層で特定したギャップ内で、さらに小さなlcmを使って探索範囲を絞り込む
- **第3層**: 同様に階層的に探索を深化

このように階層的にアプローチすることで、巨大な数の範囲でも効率的に素数候補を絞り込むことができます。

## プログラム構成

現在のバージョンは第1層のみの実装です。今後、第2層版、第3層版を追加予定です。

## 特徴

- 🎯 **非消去法アプローチ** - 消去ではなく積極的に素数候補の領域を絞り込む
- 🔢 **LCM法** - 最小公倍数を利用した革新的アルゴリズム
- 🏜️ **素数ギャップ内の探索** - 大きな素数ギャップ内の素数を効率的に発見
- 🐳 **Docker対応** - 再現可能なビルド環境
- 📊 **約1024ビット** - 教育・研究目的に適したサイズ

## アルゴリズムの原理

1. **第1層**: lcm(1,2,3,...,701) で大きな素数ギャップの位置とサイズを特定
2. **候補生成**: 特定されたギャップ内で素数候補を探索
3. **素数判定**: Miller-Rabin法で確率的素数判定

## 必要環境

- Docker Desktop
- Git

## ビルドと実行

```bash
# リポジトリのクローン
git clone https://github.com/[username]/prime-oasis.git
cd prime-oasis

# Dockerイメージのビルド
docker build -t prime-oasis .

# 実行
docker run -it prime-oasis /app/build/oasis_layer1
```

```
prime oasis = (素数の値)
prime oasis = (素数の値)

```

## 技術詳細

- **言語**: C
- **ライブラリ**: GMP (GNU Multiple Precision Arithmetic Library)
- **ビルドシステム**: CMake
- **LCM範囲**: 1..701 (約1020ビット)

## 注意事項

⚠️ **重要**: このバージョンは教育・研究目的の実装です。約1024ビットの素数を扱うため、現代の暗号システム（RSA-2048以上）には影響しません。
